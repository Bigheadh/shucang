## 核心理念

### 1. 技能沉淀
每一个经过验证的工作流、配置、脚本、经验教训，都值得被记录为一张「技能卡片」。

### 2. AI 变更透明化 🔥
每次 AI 修改代码，必须附带一份 **变更明细**，逐条列出：
- 改了哪个文件、哪一行
- 改之前是什么，改之后是什么
- 为什么改
- 可能带来的影响

详见 `术藏/模板/ai-change-log.md`

### 3. 持续迭代
技能不是死文档，用一次就要更新一次。发现更好的做法，立刻更新卡片。
